Boxes in CSS

All HTML elements are basically boxes, that can be moved and altered by CSS. The box model is a way to define and design these boxes using CSS. Using the
tag in your HTML document you can name a certain class, which you then proceed to call in CSS. Say you have a piece of text that you want style using the box model. You first wrap it in <div> tags and then write <div class=“text”>. Now you’ve turned your piece of text into a box element. Now go to your CSS file and you can access and modify the looks of your box using CSS code. You can now access the div class ‘text’ by writing for example:
div.text { background-color: yellow }

Now only the background color of the box named ‘text’ will become yellow. The background color in other boxes on your page is still whatever you defined it in the overall div {} css element.

Box models are a nice way of keeping all the elements separate from each other, so their styles don’t mix unintentionally. The tricky thing about CSS is the cascading function which cascades a set of styling rules throughout the entire element it is applied to. The box model allows you to isolate a certain part of your HTML webpage and style it only for that. In short it is a nice tool to keep total control over what comes where and how on your webpage. By creating different HTML classes it’s easy to call them in your CSS file using div.classname.
